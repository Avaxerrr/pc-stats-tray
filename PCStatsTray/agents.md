# PCStatsTray - AI Assistant Guide

This document (`agents.md`) is the project guide for future AI assistants working in `PCStatsTray`.

## 1. Project overview

- App: `PCStatsTray`
- Framework: `.NET 10` WinForms
- Hardware backend: `LibreHardwareMonitorLib`
- Optional CPU sensor backend: `PawnIO`
- Config file: `hwmon_config.json`

## 2. Hardware and runtime notes

- Some systems only expose basic CPU data without `PawnIO`.
- `CPU Load` may work while `CPU Temp`, `CPU Clock`, and `CPU Power` stay blank.
- The app now includes PawnIO guidance in the tray app and README.
- Use the official PawnIO site when pointing users to it: `https://pawnio.eu/`
- `.NET 10 Desktop Runtime` is only needed on target PCs that do not already have it installed.

## 3. Build and test commands

Standard test pass:
```powershell
dotnet test .\PCStatsTray.sln -c Release
```

Standard release build:
```powershell
dotnet build .\PCStatsTray.sln -c Release -p:UseSharedCompilation=false
```

Single-file publish:
```powershell
dotnet publish .\PCStatsTray\PCStatsTray.csproj -c Release -r win-x64 -p:PublishSingleFile=true -p:SelfContained=false
```

Notes:
- Use `-p:UseSharedCompilation=false` if `VBCSCompiler` or compiler-server file locks show up during Release builds.
- Rebuild after finishing user-visible changes so the app is ready for testing.
- Prefer the standard .NET publish output folder instead of inventing ad-hoc output paths.

## 4. Repo workflow and branch protection

- `master` is protected.
- Do not push directly to `master`.
- Create a branch, push it, and merge through a pull request.
- Required checks on `master` are:
  - `Build and test`
  - `Package Windows executable`
- Pull requests also require conversation resolution before merge.
- Force pushes and branch deletion are blocked on `master`.

## 5. Release workflow

- Pushing to `master` runs CI and packaging, but does not create a GitHub Release.
- A GitHub Release is created only when a tag like `v0.4.1` is pushed.
- Before a release tag:
  - bump version values in `PCStatsTray.csproj`
  - run Release tests/build locally
  - tag the exact release commit
- Release asset naming is generated by GitHub Actions from the tag and project version.

## 6. UI and settings notes

- OSD settings are live-applied; normal settings do not need a Save button.
- Hotkey capture fields should not auto-focus when the settings window opens.
- Desktop OSD and RTSS OSD have separate metric selections and separate hotkeys.
- Keep the tray menu wording user-friendly and explicit.

## 7. Editing guidance

- Do not revert unrelated user files.
- Be careful with untracked local files in this workspace.
- If changing README setup instructions, keep `PawnIO`, `.NET`, and `RTSS` roles clearly separated.
- If changing branch/release workflow, keep this file in sync.
